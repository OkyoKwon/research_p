## 1. 비즈니스 모델 요약

| 항목 | 내용 |
|------|------|
| **수익 유형** | SaaS형 결제 프로세서 (거래 수수료 기반) + 금융 서비스 (카드, 지급, FX) |
| **핵심 가치 제안** | 암호화폐를 수신하되 법정화폐로 정산받는 "변동성 제로" 결제 인프라 |
| **주요 고객 세그먼트** | (1) 엔터프라이즈 가맹점 (2) 중소형 온라인 가맹점 (3) 개인 소비자 (지갑/카드) (4) 대량 지급 기업 |

### 비즈니스 모델 유형 분류

BitPay는 전통 결제 프로세서(Stripe, Square)와 유사한 **양면 플랫폼 모델**을 운영한다.

- **공급 측**: 암호화폐 보유 소비자 (결제자)
- **수요 측**: 암호화폐 결제를 수신하려는 가맹점 (피결제자)
- **BitPay의 역할**: 양측 사이에서 환전 + 정산 + 컴플라이언스를 중개하며, 가맹점 측에서 수수료를 수취

## 2. 수익 구조 상세

### 2.1 수익원별 분석

#### (1) Merchant Processing Fee (핵심 수익원)

| 월간 처리량 | 수수료율 | 비고 |
|------------|---------|------|
| $500,000 미만 | 2% + $0.25/건 | 소규모 가맹점 기본 요금 |
| $500,000 -- $999,999 | 1.5% + $0.25/건 | 중간 규모 할인 |
| $1,000,000 이상 | 1% + $0.25/건 | 대형 가맹점 할인 |

- **신뢰도**: 확인됨 (공식 가격 페이지)
- **추정 수익 기여도**: 전체 수익의 60-70%
- 2024년 공식 가격 체계 개편으로 기존 단일 1% 요금에서 볼륨 기반 티어드 프라이싱으로 전환

#### (2) Network Cost (마이너 수수료 전가)

| 항목 | 내용 |
|------|------|
| **부과 대상** | 소비자 (결제자) |
| **공식 설명** | 블록체인 네트워크 수수료를 인보이스에 추가 부과 |
| **공식 기준** | 인보이스 금액의 0.05% 미만 또는 $0.01 미만이면 미부과 |
| **실제 관측** | 소액 결제 시 14-22% 수준까지 상승 사례 보고 ($15 결제에 $2.19, $10 결제에 $2.20) |

- **신뢰도**: 추정 (수수료 구조에 BitPay의 추가 마진이 번들링되어 있을 가능성)
- **분석**: Network Cost는 공식적으로 "마이너 비용 전가"로 설명되지만, 실제 온체인 네트워크 수수료 대비 현저히 높은 금액이 부과되는 사례가 다수 보고됨. 이는 BitPay가 Network Cost 명목으로 추가 수익을 올리고 있음을 시사. 특히 소액 결제에서 이 비율이 과도하게 높아지는 구조는 소비자 불만의 주요 원인

#### (3) FX 스프레드 (환전 마진)

| 항목 | 내용 |
|------|------|
| **환율 산출 방식** | 복수 승인 거래소의 매수 주문(Bids) 기반 가중 평균 |
| **법정화폐 간 환율** | openexchangerates.org 참조 |
| **환율 고정 시간** | 인보이스 생성 시점 최대 15분 |
| **시장 변동성/유동성 반영** | 환율에 시장 변동성과 유동성 깊이(market depth)를 팩터링 |

- **신뢰도**: 추정 (스프레드 크기 미공개)
- **분석**: BitPay는 "승인 거래소의 매수 주문(Bids)"을 기준으로 환율을 산정한다고 명시. Bid 가격은 시장 중간값(mid-market rate)보다 항상 낮으므로, 구조적으로 BitPay에 유리한 스프레드가 내장. 추가로 "시장 변동성과 유동성 깊이를 팩터링"한다는 표현은 변동성이 큰 시기에 스프레드를 확대할 수 있는 재량권을 의미. 이 FX 스프레드는 공식 수수료에 포함되지 않는 **은닉 수익원**으로 기능

**FX 스프레드 추정 계산**:

```
가정: 연간 결제 처리액 $1.38B (결제만), 평균 FX 스프레드 0.5-1.0%
추정 FX 수익: $6.9M -- $13.8M / 연
```

#### (4) BitPay Card 수수료

| 수수료 항목 | 금액 | 비고 |
|------------|------|------|
| 카드 발급비 | $0 | 무료 |
| 월 유지비 | $0 | 무료 |
| 국내 ATM 인출 | $2.50/건 | - |
| 해외 ATM 인출 | $2.50 + 3% | FX 수수료 3% 추가 |
| 휴면 수수료 | $5/월 | 90일 미사용 시 |
| 카드 로딩 변환 스프레드 | 미공개 (추정 0.5-1.5%) | 앱 내 환율에 마켓 스프레드 포함 |

- **신뢰도**: 확인됨 (ATM, 휴면) / 추정 (변환 스프레드)
- **현재 상태**: 2025년 2월부터 **신규 가입 중단**. Metropolitan Commercial Bank 파트너십 종료 후 새 은행 파트너 미확정 (2026년 4월 현재). 기존 카드도 대부분 비활성
- **수익 영향**: 카드 프로그램 중단으로 이 수익원은 현재 사실상 **소멸** 상태

#### (5) BitPay Send (대량 지급) 수수료

| 항목 | 내용 |
|------|------|
| **수수료율** | 1% (국내/해외 동일) |
| **운영 방식** | 기업이 법정화폐로 입금 -> BitPay가 암호화폐로 변환 -> 수령자에게 전달 |
| **지원 통화** | BTC, ETH, LTC, BCH, DOGE, SHIB, APE, MATIC, USDC, USDT, DAI, WBTC, USDP, GUSD, EUROC |
| **용도** | 급여, 마켓플레이스 정산, 계약자 지급, 리워드/리베이트 |

- **신뢰도**: 확인됨 (공식 가격 페이지)
- **추정 수익 기여도**: 전체 수익의 15-20% (페이아웃 거래 비중이 2024년 13%에서 2025년 19%로 성장)
- **분석**: BitPay Send는 법정화폐-암호화폐 변환 과정에서 1% 명시 수수료 외에 FX 스프레드에서도 추가 수익 발생 가능

#### (6) 잠재 수익원

| 수익원 | 설명 | 신뢰도 |
|--------|------|--------|
| **플로트(Float) 이자 수익** | T+1 정산 과정에서 가맹점 자금을 보유하는 동안의 이자/운용 수익 | 추정 |
| **스왑/교환 수수료** | BitPay Wallet 내 암호화폐 스왑 기능의 변환 수수료 | 추정 |
| **기프트카드 수수료** | 250+ 가맹점 기프트카드 판매 중개 수수료 (가맹점으로부터 수취) | 추정 |
| **파트너십/APM 통합 수수료** | Nuvei, BlueSnap 등 전통 결제 프로세서의 APM 통합 수수료 | 추정 |
| **HODL Pay 이자 마진** | 암호화폐 담보 대출의 금리 스프레드 (2025년 5월 출시) | 추정 |

## 3. 비즈니스 모델 캔버스

### 3.1 가치 제안 (Value Proposition)

**가맹점 대상**:
1. 암호화폐 수신 + 법정화폐 정산 = **환율 변동 리스크 제로**
2. 차지백 없는 결제 (블록체인 비가역성)
3. 글로벌 결제 수신 (233개국) + 38개국 법정화폐 정산
4. 기존 이커머스 플랫폼과 원클릭 통합
5. 회계 소프트웨어(QuickBooks/Xero) 자동 연동

**소비자 대상**:
1. 보유 암호화폐로 실물 상품/서비스 결제 가능
2. 비수탁형 지갑으로 자산 주권 보장
3. 기프트카드 구매로 간접적 암호화폐 지출 경로

### 3.2 핵심 파트너

| 파트너 유형 | 주요 파트너 | 역할 |
|------------|-----------|------|
| **은행 파트너** | 미확인 (이전: Metropolitan Commercial Bank) | 법정화폐 정산, 카드 발행 |
| **카드 네트워크** | Mastercard | 직불카드 네트워크 라이선스 |
| **블록체인 네트워크** | Bitcoin, Ethereum, Polygon, Arbitrum, Optimism, Base, Litecoin, XRP, Dogecoin 등 | 결제 트랜잭션 처리 기반 인프라 |
| **거래소 파트너** | 복수 승인 거래소 (미공개) | 환율 산출 및 유동성 공급 |

| **스테이블코인 발행사** | Circle (USDC), Tether (USDT), Gemini (GUSD) 등 | 스테이블코인 유동성 |

### 3.3 핵심 활동

1. **결제 처리**: 인보이스 생성 -> 블록체인 모니터링 -> 결제 확인 -> 정산
2. **환전/헤지**: 실시간 환율 산정, 암호화폐-법정화폐 변환, 변동성 리스크 관리
3. **컴플라이언스**: KYC/AML 심사, OFAC 제재 스크리닝, 트랜잭션 모니터링
4. **인프라 운영**: 멀티체인 블록체인 노드 운영, API/SDK 유지관리
5. **고객 온보딩**: 가맹점 심사, 통합 지원, 기술 문서 관리

### 4.4 결제 처리 아키텍처

#### API 구조

| 항목 | 상세 | 신뢰도 |
|------|------|--------|
| **API 유형** | RESTful HTTPS API | 확인됨 |
| **인증 프로토콜** | BitAuth Identity Protocol (타원 곡선 secp256k1 기반 공개키-개인키 쌍) | 확인됨 |
| **신원 확인** | SIN (System Identification Number) = RIPEMD160(SHA256(공개키)) | 확인됨 |
| **권한 모델** | Capability-based Security (Facade + Token 체계) | 확인됨 |
| **Webhook** | IPN (Instant Payment Notification), **서명되지 않음** (unsigned) | 확인됨 |
| **환율 API** | /rates 엔드포인트로 실시간 환율 조회 가능 | 확인됨 |

**인증 플로우**:
```
[클라이언트] --> (1) secp256k1 키쌍 생성
                     |
[클라이언트] --> (2) SIN = RIPEMD160(SHA256(공개키)) 도출
                     |
[클라이언트] --> (3) POST /tokens 호출 (SIN, Facade 지정)
                     |
[BitPay] --> (4) 페어링 코드 발급 --> 대시보드에서 승인
                     |
[BitPay] --> (5) API 토큰 발급 (Capability-based)
                     |
[클라이언트] --> (6) 이후 모든 API 호출에 토큰 첨부
```

> **보안 분석**: BitPay의 인증 체계는 비트코인 프로토콜과 동일한 타원 곡선 암호학을 사용하며, 이는 전통 API 키 방식보다 높은 보안성을 제공한다. 그러나 IPN(Webhook)이 서명되지 않는 점은 설계적 약점으로, 가맹점은 모든 IPN 수신 후 반드시 API로 상태를 재검증해야 한다.

#### 인보이스 시스템

| 상태 | 설명 | 전환 조건 |
|------|------|----------|
| **New** | 인보이스 생성 직후 | 생성 시 |
| **Paid** | 결제 트랜잭션 감지 (미확인) | 블록체인 멤풀 감지 |
| **Confirmed** | 기본 블록 확인 완료 | BTC: 1 confirmation |
| **Complete** | 충분한 블록 확인, 정산 크레딧 반영 | BTC: 2 confirmations |
| **Expired** | 15분 결제 창 만료 | 미결제 시 |
| **Invalid** | 결제되었으나 1시간 내 확인 실패 | 블록체인 이상 |
| **Underpaid** | 요청 금액 미달 | 자동 환불 처리 |
| **Overpaid** | 초과 결제 | 초과분 자동 환불 |

### 5.2 정산 처리 프로세스 상세

```
Phase 1: 결제 수신 및 환율 고정
=========================================
[고객] ---> 암호화폐 전송 ---> [BitPay 수신 주소]
                                     |
                              환율 고정 (Lock)
                              - 암호화폐/USD 환율: 승인 거래소 Bids 기반
                              - USD/EUR 등 법정화폐 간: openexchangerates.org
                              - 시장 변동성/유동성 팩터 적용
                                     |
                              Invoice "Paid" -> "Confirmed" -> "Complete"

Phase 2: 가맹점 계정 크레딧
=========================================
[BitPay] ---> Invoice "Complete" ---> 가맹점 대시보드 잔액 반영
              - 수수료 차감: 거래 금액 x (1-2%) + $0.25
              - Network Cost 차감: 소비자 부담 (별도)
              - 순 정산 금액 = 거래 금액 - 수수료

Phase 3: 일일 배치 정산
=========================================
[BitPay 정산 엔진] ---> 매 영업일 마감
                         - 최소 정산 잔액 확인 (USD: ~$20, EUR: 100, GBP: 1,000 등)
                         - 전일 처리 완료 건 합산
                         - 법정화폐/암호화폐 분할 비율 적용
                              |
                    +---------+----------+
                    |                    |
              법정화폐 정산         암호화폐 정산
              ACH/SEPA/Wire        지정 지갑 주소
              1-2 영업일 입금       약 1시간 내 수신

Phase 4 (내부): BitPay의 변환 처리 (추정)
=========================================
[BitPay] ---> 수신한 암호화폐를 거래소에서 법정화폐로 변환
              - 실시간 또는 배치 방식 OTC/거래소 매도
              - 변동성 헤지: 환율 고정 시점과 실제 변환 시점의 차이가 BitPay의 리스크
              - 이 리스크 프리미엄이 FX 스프레드에 반영
```

### 5.3 환율 결정 메커니즘

| 단계 | 방법 | 상세 |

## 8. 핵심 인사이트 종합

### 8.1 비즈니스 모델의 구조적 강점

1. **네트워크 효과**: 130,000 가맹점 + 260만 사용자의 양면 플랫폼 효과. 가맹점이 많을수록 소비자에게 가치 증가, 역도 성립
2. **전환 비용(Switching Cost)**: API 통합, 회계 연동, 정산 프로세스가 이미 구축된 가맹점은 전환 비용이 높음
3. **규제 해자(Regulatory Moat)**: FinCEN MSB 14년 이력, OFAC 합의 후 강화된 컴플라이언스는 신규 진입자에 대한 장벽
4. **FX 스프레드 은닉 수익**: 공식 수수료 외에 환율 스프레드에서 추가 수익을 올리는 구조는 가격 경쟁에서 명목 수수료 인하 여지를 제공

### 8.2 비즈니스 모델의 구조적 약점

1. **수탁형 모델의 이중성**: 가맹점 결제는 수탁형이나, 지갑은 비수탁형. 수탁형 결제 처리 과정에서의 자금 관리 투명성 부족
2. **은행 의존성**: 법정화폐 정산의 핵심 가치가 은행 파트너십에 의존하며, Metropolitan Commercial Bank 철수 사례가 이 리스크를 증명
3. **소비자 서비스의 열위**: B2B 결제 인프라 기업이 B2C 서비스(Wallet, Card)까지 확장하면서 자원 분산 및 소비자 경험 저하
4. **오픈소스 패러독스**: Bitcore를 오픈소스로 공개하면서 BTCPay Server 같은 무료 대안의 기술적 기반을 간접적으로 제공

### 8.3 기술적 차별점과 한계

**차별점**:
- 자체 개발 블록체인 풀스택(Bitcore)을 14년간 운영/유지한 기술 축적
- BitAuth(secp256k1 기반 인증)은 업계에서 독자적인 보안 프로토콜
- 7개 언어 SDK와 6+ 이커머스 플러그인의 폭넓은 개발자 생태계

**한계**:
- Lightning Network 미지원으로 소액 결제 시장 진입 불가
- IPN 미서명은 보안 관행상 구식 (Stripe, PayPal은 Webhook 서명 기본 제공)
- 15분 결제 창은 네트워크 혼잡 시 UX 저하의 구조적 원인
- L2 네트워크(Arbitrum, Optimism, Base)는 지갑에서만 지원하고 결제에는 미지원
