## 슬라이드 #3: Executive Summary -- 전체 핵심 인사이트
- **레이아웃**: 텍스트 (4분할)
- **제목**: Executive Summary -- 전체 핵심 인사이트
- **본문**:
  - **Base Pay**: 업계 유일 온체인 에스크로 기반 Authorize-Capture 프로토콜. Coinbase 생태계(6,000만+ 사용자) + Shopify(550만 가맹점) + x402 AI 결제(연간화 $600M). **약점: 법정화폐 직접 정산 미지원**
  - **Binance Pay**: 누적 거래량 **$280B+**, 2,000만 가맹점의 세계 최대 암호화폐 결제 플랫폼. 오프체인 10,000+ TPS 즉시 정산. B2C 결제의 98%가 스테이블코인. **약점: 법정화폐 정산 불가, 70개국+ 미진출**
  - **Coinbase Commerce**: 시장점유율 12%(3위). Base L2 + Shopify 파트너십. 결제-정산 비용 우위(1% vs 전통 2.9%). **약점: 환불 프로세스 업계 최하위, 2026.03 Commerce 종료**
  - **Stripe Crypto**: Bridge($1.1B 인수) + Tempo + Privy 수직 통합으로 "스테이블코인 풀스택 클라우드" 구축. 가맹점 크립토 노출 완전 제거. **약점: 미국 전용 수취, 1.5% 프리미엄**
- **스피커노트**: 4개 서비스 모두 결제 수수료는 1~1.5%로 전통 카드 결제(2.9%+$0.30)보다 저렴합니다. 그러나 법정화폐 자동 정산은 Stripe만 제공하며, 이것이 메인스트림 채택의 최대 결정 요인입니다.

## 슬라이드 #11: [Base Pay] 결제 시나리오 A: 즉시 결제 (Charge)
- **레이아웃**: 다이어그램 + 테이블
- **제목**: [Base Pay] 결제 시나리오 A: 즉시 결제 (Charge)
- **본문** (Step-by-step):
  1. 머천트가 결제 요청 생성 (API 또는 Shopify 플러그인)
  2. 오퍼레이터가 TransferIntent 생성 (금액, 머천트 주소, 만료 기한, 수수료)
  3. 구매자가 지갑 연결 후 결제 승인
  4. 스마트 컨트랙트 실행 (자동 스왑: Uniswap V3 + Permit2/EIP-3009)
  5. 머천트 지갑에 정산 토큰 직접 전송 + 수수료 차감
  6. Transferred 이벤트 발행 (약 **2초**)
- **테이블** (수수료):
  | 항목 | 금액 | 부담 주체 |
  |------|------|-----------|
  | 플랫폼 수수료 | 1% | 머천트 (정산에서 차감) |
  | 네트워크 가스비 (Base) | ~$0.01 | 구매자 |
  | DEX 스왑 슬리피지 | 변동 | 구매자 (토큰 다를 경우) |
- **본문**: 적합 사용처 -- 디지털 상품, 구독 활성화, 에스크로 불필요 시나리오
- **스피커노트**: 즉시 결제는 디지털 상품이나 즉시 배송이 필요한 시나리오에 적합합니다. 구매자 토큰이 머천트 정산 토큰과 다르면 Uniswap V3를 통해 자동 스왑됩니다.

## 슬라이드 #12: [Base Pay] 결제 시나리오 B: 인가-캡처 결제 (Authorize-Capture)
- **레이아웃**: 다이어그램 + 테이블
- **제목**: [Base Pay] 결제 시나리오 B: 인가-캡처 결제
- **본문** (Step-by-step):
  1. PaymentInfo 생성 (maxAmount, authorizationExpiry, refundExpiry, feeBps)
  2. 구매자 결제 승인 (지갑 서명)
  3. **Authorize**: Token Collector → 에스크로(Token Store)에 예치
  4. **[대기 기간]**: 주문/재고/배송 확인, Void 가능
  5. **Capture**: 에스크로 → 머천트 (**부분 캡처 지원**)
  6. 미캡처 잔여분: 자동 반환 또는 reclaim()
- **본문** (에스크로 메커니즘):
  - Minimal Proxy 패턴으로 가스 효율 최적화
  - 오퍼레이터별 격리 (자금 혼합 방지)
  - 비업그레이드 컨트랙트 (배포 후 변경 불가)
  - ReentrancyGuard 내장
- **테이블** (수수료):
  | 항목 | 금액 | 시점 |
  |------|------|------|

- **스피커노트**: Charge와 Auth-Capture 두 가지 모델을 모두 지원합니다. Base Pay와 동일한 프로토콜을 공유하지만, Coinbase Commerce는 기존 Commerce API와의 호환성도 유지하고 있습니다.

  4. **AI 에이전트 결제가 차세대 전장**
     - Stripe(3중 전략)과 Base Pay(x402)가 선점 경쟁 중. 90일 내 6개 프로토콜 론칭
  5. **운영 품질(고객 지원, 환불)이 기술 혁신만큼 중요**
     - 4개 서비스 중 3개에서 고객 지원이 최대 불만. 기술적 우위가 운영 품질을 따라가지 못하는 공통 과제
- **스피커노트**: 이 5가지 핵심 메시지가 본 리서치의 최종 결론입니다. 크립토 결제의 미래는 기술 혁신과 운영 품질의 균형에 달려 있으며, 법정화폐 정산과 AI 에이전트 결제가 향후 시장 판도를 결정할 것입니다.
